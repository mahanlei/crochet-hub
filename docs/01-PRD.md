# CrochetHub — 钩织作品与教程分享平台 PRD

## 1. 项目背景

钩织（Crochet）是一种受众广泛的手工艺爱好。创作者在完成作品后，往往希望记录制作过程、分享教程、展示成品。目前市面上缺少一个轻量、专注于钩织领域的作品展示与教程分享平台。

CrochetHub 旨在为钩织爱好者提供一个简洁美观的线上空间，用于展示作品、分享分步教程，并借助 AI 能力辅助生成作品描述与材料推荐，降低内容创作门槛。

## 2. 目标用户

- **创作者**：希望记录和展示自己钩织作品、分享制作教程的手工爱好者
- **浏览者**：对钩织感兴趣，想欣赏作品、学习教程的用户

## 3. 项目目标

- 提供作品展示与教程分享的完整用户体验
- 集成 AI 能力，辅助生成作品描述与材料推荐
- 两天内完成 MVP，具备可演示的线上版本

## 4. MVP 功能边界

### 4.1 包含的功能（MVP 范围内）

#### 前台展示

| 功能 | 说明 |
|------|------|
| 首页作品瀑布流 | 以卡片形式展示所有作品，含封面图、标题、工艺类别标签、难度标签 |
| 作品详情页 | 多图轮播展示成品，显示材料清单、工具列表、分步教程 |
| 分类筛选 | 支持按工艺类别（钩针 / 棒针）和难度等级（入门 / 进阶 / 高级）快速筛选作品 |
| 响应式布局 | 适配桌面端与移动端浏览 |

#### 后台管理

| 功能 | 说明 |
|------|------|
| 作品列表管理 | 查看、编辑、删除已有作品 |
| 新建作品 | 填写标题、描述、工艺类别、难度、材料清单，上传封面图与步骤图 |
| 教程步骤管理 | 为每个作品添加分步教程（步骤序号 + 图片 + 文字说明） |
| AI 辅助生成 | 上传作品图片后，AI 自动生成作品描述与材料推荐，用户可编辑 |

### 4.2 不包含的功能（后续迭代）

- 用户注册 / 登录体系（MVP 阶段后台无需鉴权，专注内容管理）
- 评论 / 点赞 / 收藏
- 搜索功能
- 多语言支持

## 5. 页面列表

| 页面 | 路由 | 说明 |
|------|------|------|
| 首页 | `/` | 作品瀑布流展示，支持按工艺类别与难度筛选 |
| 作品详情 | `/works/:id` | 作品信息 + 多图轮播 + 分步教程 |
| 后台 - 作品列表 | `/admin` | 作品管理列表（增删改） |
| 后台 - 新建/编辑作品 | `/admin/works/new` `/admin/works/:id/edit` | 作品表单 + 图片上传 + AI 生成 |

## 6. 交互说明

### 6.1 首页

用户打开首页，看到作品卡片瀑布流。每张卡片展示封面图、标题、工艺类别标签（钩针/棒针）和难度标签（以颜色区分：绿色=入门，橙色=进阶，红色=高级）。点击卡片进入详情页。页面顶部有筛选标签栏，分为两组：工艺类别（全部/钩针/棒针）和难度等级（全部/入门/进阶/高级），两组可组合筛选。

### 6.2 作品详情页

顶部为作品多图轮播区域，下方分为两个板块：左侧（或上方）展示作品基本信息（标题、描述、工艺类别、难度、材料清单、工具列表），右侧（或下方）展示分步教程，每步包含步骤图和文字说明。

### 6.3 后台 - 新建作品

表单包含：标题（文本输入）、描述（多行文本，旁边有"AI 生成"按钮）、工艺类别（单选：钩针/棒针）、难度（下拉选择）、材料清单（可动态增减的输入项）、工具列表（可动态增减）、封面图上传、教程步骤（可排序的步骤列表，每步含图片上传和文字说明）。

点击"AI 生成"按钮时，系统将封面图发送给 AI，返回建议的描述文本和材料推荐，用户可直接采用或修改。

## 7. 非功能性要求

- **性能**：首页图片使用懒加载，接口响应时间 < 500ms
- **可部署**：前端可部署至 Vercel，后端可部署至 Railway / Render
- **代码规范**：ESLint + Prettier，清晰的目录结构与注释
